1. Problem Statement & User Research
Schools produce a relentless stream of communications — emails, LMS posts, newsletters, PDF forms,
Google Classroom announcements. Inside them are deadlines, permission slips, scholarship links, and
required actions. For most students, filtering signal from noise is merely annoying. For ADHD students with
executive dysfunction, it is cognitively near-impossible — a documented neurological barrier, not a character
flaw.
"Every week, an ADHD student opens a school email and misses something critical
— not because they don't care, but because finding the action item inside 500 words
of text is a task their brain makes nearly impossible."

2. AI Architecture — 7-Stage Gemini Pipeline
ActionPath v2 replaces the original 5-stage Claude pipeline with a fully AI-powered 7-stage Gemini pipeline.
Every single stage — including the pre-processing classifier — makes a Gemini API call. No rule-based
fallbacks. No heuristics masquerading as AI. Each agent has one focused job, a defined JSON output
schema, and is assigned to the most cost-efficient Gemini model for that task.
STAGE 1
INTAKE CLASSIFIER AGENT
I gemini-3.1-flash-lite
Receives raw input (pasted text, .txt, .pdf, .eml, .html). Classifies communication type: deadline_notice
/ general_update / emergency / form_request / payment_required / event_invitation / policy_change.
Estimates cognitive_load_score (0–10) based on word count, sentence complexity, action density, and
urgency language. Chooses the optimal downstream prompt template. Returns a routing decision that
controls all subsequent agents.
Output schema:
{ "comm_type": "deadline_notice",
"cognitive_load_score": 7.4,
"word_count": 412,
"action_density": 0.08,
"routing_template": "DEADLINE_V2",
"language_complexity": "high" }
STAGE 2
ENTITY & CONTEXT EXTRACTION AGENT
I gemini-3.1-pro-preview
Full deep extraction using Gemini 2.5 Pro — the most capable model, used here because extraction
accuracy is critical and errors cascade to all downstream stages. Extracts: every task/deadline/action
item, the exact source sentence, people mentioned, amounts/fees, URLs, form names, room numbers,
and subject associations. Outputs structured JSON with span-level source linking.
Output schema:
"items": [{ "task": "...", "deadline": "2026-06-17",
"source_sentence": "exact quote...",
"item_type": "form|deadline|payment|meeting|action",
"people": ["Ms. Rivera"], "url": null,
"subject": "Biology", "confidence": 0.94 }]

--- PAGE 6 ---
STAGE 3
SCORING & COLLISION AGENT
I gemini-3.5-flash
Takes extraction output. Scores each item on four axes: urgency (days until deadline, 1–10), effort (low
= click / medium = fill form / high = write / very_high = multi-step project), dependency (does this block
or require another item), and consequence_severity (what happens if missed: grade / opportunity /
social / financial). Detects deadline collisions (3+ tasks on same day) and suggests reordering. Outputs
a ranked priority list.
Output schema:
"scored_items": [{ "task_id": "t1",
"urgency": 9, "effort": "low",
"consequence": "grade", "dependency": null,
"composite_score": 8.7 }],
"collision_days": ["2026-06-17"],
"reorder_suggestion": "Move lab fee to today"
STAGE 4
CONTEXT FUSION AGENT
I gemini-3.1-flash-lite
Merges the scored items with the student's stored context: completed tasks, known subjects, past
deadline patterns, and preferred working times. Deduplicates items that match previously seen tasks.
Flags items that are recurring (weekly newsletter format detected). Lightweight — uses Flash-Lite
because the reasoning is straightforward merge-and-deduplicate logic, not creative reasoning.
Output schema:
"merged_items": [...],
"deduplicated": ["t3 matches seen on 2026-06-10"],
"recurring_detected": true,
"student_subject_match": { "Biology": ["t1", "t4"] }

--- PAGE 7 ---
STAGE 5
DAY-BY-DAY PLANNING AGENT
I gemini-3.5-flash
Takes merged scored items. Generates a concrete daily action plan across three horizons: Today /
Tomorrow / This Week / Later. Applies cognitive load balancing — never assigns more than 3
high-effort tasks to a single day. If a day is overloaded, suggests which items to move and why.
Generates an estimated time budget per day (e.g. 'Today: ~45 minutes total'). Accounts for the
student's stored 'busy days' preference.
Output schema:
"plan": {
"today": [{"task_id":"t1","est_mins":5}],
"tomorrow": [{"task_id":"t2","est_mins":20}],
"this_week": [...], "later": [...] },
"load_warning": "Wednesday has 4 items (42 min total)",
"daily_budgets": {"today": 25, "tomorrow": 20}
STAGE 6
LANGUAGE REWRITER AGENT
I gemini-3.5-flash
Rewrites every task description into plain, calm, ADHD-optimized language. Rules: max 12 words per
task, active voice, no bureaucratic phrasing, no passive constructions, start with a verb. Adds a
one-sentence 'why this matters' if the task has a grade/opportunity consequence. Also generates a
'start cue' — the single first physical action the student should take (e.g. 'Open Google Forms → ...').
This removes the initiation barrier.
Output schema:
"rewritten": "Sign field trip form. Hand to room 204.",
"why_it_matters": "Missing this means you cannot attend.",
"start_cue": "Open your backpack and find the blue form.",
"original": "Please return the signed permission slip..."

--- PAGE 8 ---
STAGE 7
QUALITY ASSURANCE AGENT
I gemini-3.1-flash-lite
Final pass QA agent — new in v2. Checks the full pipeline output for: missing deadlines (compares
extracted count vs. a re-scan), confidence < 0.7 (flags for student review), any task with no start_cue
(requests regeneration), overloaded days not flagged with warnings, and any rewritten task over 12
words. Returns a qa_passed boolean and a list of qa_issues for the UI to surface.
Output schema:
"qa_passed": false,
"qa_issues": [
{ "task_id": "t3", "issue": "low_confidence",
"action": "flag_for_review" },
{ "task_id": "t5", "issue": "rewrite_too_long" }
]

10. Devpost Submission Copy
Project Description
ActionPath is an AI-powered executive function tool for ADHD high school students who struggle with
deadlines buried inside school communications. Students paste or upload a school email, LMS post, or
PDF. A 7-stage Gemini AI pipeline — powered entirely by Gemini 2.5 Pro, Gemini 2.5 Flash, and Gemini 2.0
Flash-Lite — classifies the communication, extracts every deadline and required action with span-level
source linking, scores each item by urgency/effort/consequence, fuses with the student's context, generates
a day-by-day action plan with time budgets, rewrites every task in plain language under 12 words with a start
cue, and QA-checks its own output. The result is an interactive checklist with a week view, cognitive load
warnings, confidence badges, start cue cards, and one-click calendar export. AI recommends. Student
decides and acts.
